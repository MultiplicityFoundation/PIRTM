# v0.1.0 Migration Notes (Draft)

Status: draft  
Target release: v0.1.0  
Date: YYYY-MM-DD

## Audience

This guide is for users upgrading from pre-`v0.1.0` development snapshots to the `v0.1.0` release line.

## Summary of User-Facing Changes

- Transpiler CLI JSON payload defaults changed to reduce noisy output.
- ACE/certification surfaces are moving toward ACE-native certificate contracts.
- Legacy module boundary remains available but explicitly deprecated and scheduled for policy finalization.

## CLI Behavior Changes

### Transpiler output gating is now explicit

By default, transpiler JSON output omits:

- `witness_json`
- `lambda_events`

To include them, pass explicit flags:

- `--emit-witness`
- `--emit-lambda-events`

#### Before (pre-`v0.1.0` behavior may include fields implicitly)

```bash
python -m pirtm transpile --output json ...
```

#### After (`v0.1.0` and later)

```bash
python -m pirtm transpile --output json --emit-witness --emit-lambda-events ...
```

### Metadata parsing is stricter

`--metadata` must be valid JSON and decode to an object. Invalid JSON or non-object payloads now fail with explicit CLI errors.

## API and Contract Notes

### Certificate migration path

`pirtm.certify` compatibility is documented in `docs/migration/certify-v1.md`.

Key direction:

- Prefer ACE-native certificate contracts for new integrations.
- Legacy certificate compatibility shims remain available for transition periods.

## Deprecations and Timeline Signals

- `_legacy` package surfaces are deprecated and remain transition-only.
- Module-level deprecation timeline decisions are tracked in roadmap issues `R4` and `R5` and will be finalized before/with follow-up release planning.

## Action Checklist for Adopters

- Audit transpiler consumers for assumptions that `witness_json` and `lambda_events` are always present.
- Update automation/scripts to pass `--emit-witness` / `--emit-lambda-events` when these payloads are required.
- Review certification-type usage and align new code with ACE-native certificate contracts.
- Track roadmap decisions for `_legacy` boundary and sunset policy.
