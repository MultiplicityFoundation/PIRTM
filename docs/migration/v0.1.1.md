# Migration Guide: v0.1.0 → v0.1.1

## Certificate API Clarification

`contraction_certificate()` is now the primary API for contraction validation.
`ace_certificate()` remains available for ACE-native diagnostics.

## Migration

Before:

```python
from pirtm import ace_certificate
cert = ace_certificate(info)
```

After (recommended):

```python
from pirtm import contraction_certificate
cert = contraction_certificate(info)
```

Advanced ACE diagnostics remain available:

```python
from pirtm import ace_certificate
ace = ace_certificate(info)
```

## Compatibility

No breaking changes are introduced for existing `v0.1.0` code.

## Deprecations

- `legacy_ace_certificate()` → use `contraction_certificate()`
- `ace_certificate_v2()` → use `ace_certificate()`
